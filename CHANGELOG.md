# CHANGELOG

## v0.1.2 - 2026-03-08

本次版本主要围绕 reviewer 新一轮意见展开。

### Fixed

- 一些小问题

## v0.1.1 - 2026-03-08

本次版本主要围绕 reviewer 评论、AstrBot 插件开发文档对齐、配置持久化整理和运行稳定性修复展开。

### Added

- 新增 `搜jm` / `看jm` 双主链路交互
- 新增搜索结果封面缩略图拼图预览
- 新增搜索结果转发消息发送
- 新增搜索结果前若干条作品总页数补全显示
- 新增多章节详情受控并发补全章节标题与页数
- 新增 `看jm [序号或id] [章节] P[页码]` 单图提取能力
- 新增文本消息与转发消息自动撤回配置
- 新增页数补全缓存 TTL 与容量上限控制
- 新增 `jm_option.yml` 持久化读写分层结构

### Changed

- 统一插件版本号为 `v0.1.1`
- 配置加载逻辑改为优先读取 `_conf_schema.json` 默认值
- `load_config()` 改为默认只读，不再每次加载都无条件写回 `config.yaml`
- 搜索缓存键从单纯 `sender_id` 调整为会话级 `session_key`
- `jm更新域名` / `jm清空域名` 调整为稳定持久化配置流程
- `jm_option` 构造改为运行时读取持久配置，不再频繁重建覆盖文件
- 清理运行时缓存时不再误删 `jm_option.yml`
- README 文档整体重写，和当前实现保持一致

### Fixed

- 修复 logger 使用不规范问题
- 修复阻塞调用直接运行在事件线程中的问题
- 修复搜索缓存跨群/跨私聊串用问题
- 修复 `_page_count_cache` 无上限增长问题
- 修复全局代理配置在多线程下的竞态问题
- 修复共享 client 带来的线程安全隐患
- 修复临时 `jm_option_xxx.yml` 文件泄漏问题
- 修复域名更新/清空后可能被后续运行逻辑覆盖的问题
- 修复 `config_manager.py` 中部分默认值未严格对齐 `_conf_schema.json` 的问题
- 修复版本信息在 `metadata.yaml` 与 `@register(...)` 之间不一致的问题
- 修复运行时清理逻辑会误删持久域名配置的问题

### Removed

- 删除历史残留死代码与无效配置映射
- 删除过时的 `output_batch_size` 映射
