# Changelog

本文档记录 `astrbot_plugin_jm_bot` 在本次发布前整理阶段的主要改动。

## Unreleased

### Added

- 新增 `搜jm` / `看jm` 双入口工作流，分别承担搜索预览与详情下载
- 新增搜索结果封面缩略图拼图预览
- 新增搜索结果转发消息发送
- 新增搜索结果页前若干条作品总页数补全显示 `XXP`
- 新增多章节详情受控并发补全章节标题与页数
- 新增 `看jm [序号或id] [章节] P[页码]` 单图提取能力
- 新增文本消息与转发消息自动撤回配置支持
- 新增统一运行缓存清理命令 `jm清理缓存`
- 新增 README 中的 Docker 多容器文件互通说明

### Changed

- 将原先收口后的旧 JM 指令入口与过渡提示移除，统一为 `搜jm` / `看jm`
- 命令注册方式调整为支持从配置中修改别名，并保留默认命令别名
- `jm更新域名` 与 `jm清空域名` 默认改为管理员权限执行
- 搜索页封面获取流程重构为基于封面缓存目录的稳定下载与复用
- 搜索结果发送体验调整为更接近 `astrbot_plugin_ehentai_bot` 的转发消息风格
- PDF 生成流程移入后台线程，并通过串行锁控制并发构建，降低 AstrBot 主线程阻塞风险
- 章节列表在达到折叠阈值时改为转发消息发送，改善长列表展示体验
- 配置加载逻辑整理为真实生效的嵌套配置结构，删除无效或误导性的旧配置项
- README 补充安装、配置、依赖、Docker 部署与行为管理说明，便于直接发布

### Fixed

- 修复命令执行时可能误触发 LLM 的问题
- 修复 `看jm` 流程中 `_send_pdf_files` 缺失导致的章节读取失败
- 修复 `看jm 3` 一类流程中由异步生成器误用引发的异常
- 修复 `base_message is not defined` 导致的章节读取失败
- 修复搜索页缩略图偶发无法显示的问题
- 修复封面缓存未正确生成、未命中时下载不稳定的问题
- 修复封面下载链路中遗漏 `import asyncio` 导致的整体失效
- 修复并发任务共同写入同一 `jm_option.yml` 时造成的配置损坏问题
- 修复 `_conf_schema.json` 格式与默认值异常问题，避免插件加载失败或配置误导
- 修复 `jm清理缓存` 清理范围不足的问题，确保下载目录、封面缓存、缓存文件、运行时临时文件可统一清理
- 修复残留 `requests` 死代码与文档说明不一致问题

### Removed

- 删除旧的 JM 指令入口及相关过渡提示
- 删除无效配置项与未真正生效的伪参数
- 删除排障阶段临时加入的调试日志
- 删除不应提交到仓库的运行缓存说明残留，发布时应忽略 `__pycache__/` 与 `*.pyc`

### Notes

- `aiohttp` 由 AstrBot 运行环境提供，因此未单独写入插件 `requirements.txt`
- Docker 部署时，如果 `AstrBot` 与 `NapCat` 处于不同容器，请确保双方能够访问同一份文件路径；否则图片、PDF 或转发消息中的图片节点可能发送失败
- 示例路径：

```text
/opt/1panel/apps/astrbot/astrbot/data
/AstrBot/data
```

如果这些路径并非同一份实际挂载，请通过统一挂载点、软链接或其他映射方式打通
